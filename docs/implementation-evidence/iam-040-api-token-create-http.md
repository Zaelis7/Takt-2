Valid requests reach exactly one typed port with a stable cross-architecture SHA-256 request hash and return `201`, `Location`, `ETag: "1"`, `Cache-Control: no-store` and the one-time token response for either Create or identical replay.

The token is serialized only after the returned projection and all response headers pass fail-closed validation. Authentication, CSRF, permission, validation, idempotency-conflict and infrastructure errors map to bounded stable Problems without reflecting inputs or secrets; internal logging contains only a fixed event code and request ID.

- Data and observability review: no database, audit or external-data-flow behavior changed. The boundary invokes one port and never emits an audit event itself, so production composition can preserve the existing atomic business-write/audit/replay invariant without duplication. No secret is logged, included in a Problem, or exposed by `Debug`.
